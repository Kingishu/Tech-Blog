C#核心概念深度解析

📅 2025-09-26 🏷️ C#, .NET, Unity ⏱️ 阅读时间：7分钟

本文档旨在深入探讨C#编程语言中的几个核心且基础的概念，包括转义字符、类型转换、枚举、接口的显式实现，以及多态中的类型判断和 `new` 关键字的使用。理解这些基础知识对于构建健壮、可维护的应用程序至关重要。

一、转义字符

在字符串中，某些字符具有特殊含义。为了按字面意义使用它们，我们需要使用转义字符，即反斜杠 `\`。

常用的转义字符

  • \': 表示单引号。例如: string str = "It\'s a cat.";
  • \": 表示双引号。例如: string str = "他说:\"你好\"";
  • \n: 表示换行符。
  • \\: 表示一个反斜杠本身。例如: string path = "C:\\Windows\\System32";

不常用的转义字符

  • \t: 制表符，相当于按下Tab键，通常用于对齐文本。
  • \b: 光标退格，将光标位置向后移动一个字符。
  • \0: 空字符，在某些底层操作或字符串结尾处理时会用到。
  • \a: 警报音，执行时会让系统发出一声提示音。

取消转义

如果你希望字符串中的所有字符都按其字面意义显示，可以在字符串前面加上 @ 符号。这在处理文件路径时特别有用。

// 使用@可以避免转义，代码更清晰
string path = @"C:\Users\Documents\MyFile.txt";

二、类型转换

在C#中，将一种数据类型的值转换为另一种数据类型是常见的操作。

隐式类型转换

从小范围类型到大范围类型的转换（例如 `int` 到 `long`），编译器可以自动完成，因为这个过程不会丢失数据精度。这被称为隐式转换。

int myInt = 100;
long myLong = myInt; // 隐式转换，安全

显式类型转换（强制转换）

当转换可能导致数据丢失时（例如 `double` 到 `int`），需要显式地告诉编译器你意图进行此转换。

  1. 括号强转: 简单直接，但如果转换失败会在运行时抛出 `InvalidCastException`。double d = 9.78; int i = (int)d;
  2. Parse方法: 主要用于将字符串转换为数值类型。例如 int.Parse("123")。如果字符串格式不正确，会抛出异常。
  3. Convert类: 提供了一系列方法，可以在多种类型之间进行转换，例如 Convert.ToInt32("123")。它比Parse更灵活，可以处理 `null` 值（通常返回0）。
  4. ToString()方法: 几乎所有类型都具备此方法，用于将其值转换为字符串表示。例如 int num = 10; string s = num.ToString();

三、枚举（Enum）

枚举是一组命名的整型常量，它增强了代码的可读性和类型安全。它的本质是整数类型的包装。

在编译后，枚举类型会被编译为一个继承自 `System.Enum` 的密封类（sealed class），因此枚举是值类型。每个枚举成员本质上是一个常量整数，默认从 0 开始递增。

public enum Direction { North, South, East, West }
// North = 0, South = 1, East = 2, West = 3

上面的定义在编译后，其概念等价于：

public sealed class Direction : System.Enum
{
    public const int North = 0;
    public const int South = 1;
    public const int East = 2;
    public const int West = 3;
}

四、接口（Interface）与显式实现

接口定义了一个契约。任何实现该接口的类或结构都必须提供接口中所定义成员的实现。常规的接口实现是直接在类中定义一个同名的 `public` 方法。

当方法名冲突时

当一个类需要实现两个包含同名方法的接口时，常规实现方式会导致冲突。例如：

interface ICanFastRun { void Run(); }
interface ICanSlowRun { void Run(); }

为了解决这个问题，我们需要使用显式接口实现。

public class Dog : ICanFastRun, ICanSlowRun
{
    void ICanSlowRun.Run()
    {
        Console.WriteLine("我可以跑的很慢");
    }
    void ICanFastRun.Run()
    {
        Console.WriteLine("我可以跑的很快");
    }
}
显式接口实现的成员默认是 `private` 的，不能通过类的实例直接调用。它只能通过接口类型的引用来调用。

这种机制一方面解决了命名冲突，另一方面也起到了隐藏方法、清洁API的作用。调用时必须将对象强制转换为对应的接口类型：

Dog dog = new Dog();
// dog.Run(); // 这行代码会编译报错！

// 必须强制类型转换
((ICanFastRun)dog).Run(); // 输出: 我可以跑的很快
((ICanSlowRun)dog).Run(); // 输出: 我可以跑的很慢

五、多态中的类型

在面向对象编程中，一个对象变量可以引用不同类型的对象。我们需要区分变量的“编译时类型”和“运行时类型”。

public class Animal { }
public class Dog : Animal { }
public class Cat : Animal { }

class Program
{
    static void Main(string[] args)
    {
        Animal animal = new Animal();
        Animal dog = new Dog(); // 子类实例赋值给父类引用
        Animal cat = new Cat();
    }
}

在上面的代码中，`animal`, `dog`, `cat` 这三个变量的类型是什么？

  • 编译时类型: 编译器在检查语法和类型安全时所看到的类型。这三个变量的编译时类型都是 `Animal`，因为它们都是这样声明的（等号左边）。
  • 运行时类型: 程序实际运行时，变量所引用的对象的真实类型。`animal` 的运行时类型是 `Animal`，`dog` 的运行时类型是 `Dog`，`cat` 的运行时类型是 `Cat`（由 `new` 关键字决定，即等号右边）。

六、多态中的 new 关键字

`new` 关键字用于在派生类中隐藏基类的成员。它会中断继承链中的虚方法调用，从而影响多态行为。

public class Animal
{
    public virtual void Eat()
    {
        Console.WriteLine("我正在吃");
    }
}

public class Dog : Animal
{
    public new void Eat() // 使用 new 隐藏了基类的 Eat 方法
    {
        Console.WriteLine("狗正在吃");
    }
}

class Program
{
    static void Main(string[] args)
    {
        Animal animal = new Animal();
        Animal dog1 = new Dog();
        Dog dog2 = new Dog();

        animal.Eat();
        dog1.Eat();
        dog2.Eat();
    }
}

输出结果为：

我正在吃
我正在吃
狗正在吃

为什么 `dog1.Eat()` 输出 "我正在吃"？

关键在于 `new` 关键字切断了 `Animal.Eat` 和 `Dog.Eat` 之间的多态联系。

  1. 编译阶段：编译器只看变量的声明类型。`dog1` 的声明类型是 `Animal`。编译器检查到 `Animal` 类有 `Eat()` 方法，编译通过。由于 `Dog.Eat()` 是用 `new` 声明的，编译器认为它是一个与基类无关的全新方法，因此 `dog1` 的调用被绑定到了 `Animal.Eat()`。
  2. 运行阶段：CLR（公共语言运行时）执行编译时已经确定的调用。因为 `dog1` 在编译时被绑定到 `Animal.Eat()`，所以即使 `dog1` 的实际对象是 `Dog`，它仍然执行基类的 `Eat()` 方法。相比之下，`dog2` 的声明类型是 `Dog`，所以它自然调用 `Dog` 类中的 `Eat()` 方法。
简单总结：当子类方法使用 new 关键字时，调用哪个方法取决于变量的编译时类型（声明类型）；而当子类方法使用 override 关键字时，调用哪个方法取决于变量的运行时类型（实际实例类型），这才是真正的多态。
